## 一、项目概述

### 1.1 项目背景

广西边境线长达1020公里，与越南接壤，是我国面向东盟开放合作的前沿和窗口。然而，复杂的边境环境给生态保护和打击走私带来了巨大挑战：

- **野生动物走私猖獗**: 穿山甲等濒危物种被非法猎捕、走私
- **生态监测困难**: 传统手段难以实现全天候、立体化监测
- **执法效率低下**: 预警信息分散，处置流程冗长
- **协同作战不足**: 各部门信息孤岛，难以形成合力

### 1.2 项目定位

**热眼擒枭**定位为一款专为边境一线打造的**智能防控与生态协同预警平台**。

项目以"生态警务"为统领，立足环境资源与食药侦查职能，聚焦广西边境口岸等复杂地带的实战需求，实现：

- 边境活物走私的**精准识别**
- 生态环境的**协同监测**
- 执法任务的**高效处置**
- 证据链的**防伪溯源**

### 1.3 项目愿景

> **"热眼"识万物，"擒枭"守边境**

利用人工智能和多模态感知技术，打造一双洞察一切的"热眼"，精准识别走私行为和生态异常，让违法行为无处遁形。

### 2.2 核心技术亮点

#### 亮点一：多模态融合感知

- **红外热成像**: 检测生物体温特征，夜间识别率提升40%
- **振动光纤围栏**: 精准感知围网震动，误报率降低60%
- **可见光摄像**: 视频智能分析，支持物种自动识别
- **多源数据融合**: 时空一致性校验，准确率提升至94.5%

#### 亮点二：AI智能研判

基于深度学习的多模态时空融合算法：

```
置信度评分 = f(红外信号强度, 震动频率, 可见光特征, 时空一致性)
```

- 置信度 ≥ 85: 自动推送紧急预警
- 置信度 70-84: 进入人工复核队列
- 置信度 < 70: 仅记录存档

#### 亮点三：防伪取证系统

证据防伪校验码格式：

```
EV-{随机HASH}-{时间戳}
```

校验内容：
- 图片/视频完整性（哈希值校验）
- 采集时间（不可篡改）
- GPS坐标（精确到米）
- 采集人员身份

#### 亮点四：离线韧性设计

- **断点续传**: 网络中断后自动缓存，恢复后自动上传
- **离线队列**: 支持离线状态下完成任务处理
- **本地校验**: 离线时仍可进行证据完整性校验

## 八、团队分工

本项目采取"四组协同"模式，围绕"走私活物防控 + 环境监测"核心方向展开分工，各组职责明确、接口清晰，形成感知→算法→开发→硬件→设计的完整闭环。

---

### 8.1 算法组

**核心目标**：优化影子追踪 V2.0 与多模态数据融合算法，适配低成本硬件，保证综合识别准确率 > 85%。

| 工作项 | 具体内容 | 交付物 |
|-------|---------|--------|
| 影子追踪 V2.0 | 在 V1.0 基础上引入时序记忆网络，提升连续跟踪稳定性，解决目标短暂遮挡后身份跳变问题 | 跟踪算法代码 + 评测报告 |
| 多模态融合 | 将红外热像、振动信号、可见光三路特征统一映射到融合向量空间；设计置信度校准层，降低低置信误报 | 融合模型权重 + 消融实验结果 |
| 低成本适配 | 对模型进行 INT8 量化 + 剪枝，推理延迟控制在边缘设备可接受范围（≤ 200ms/帧） | 量化后模型 + 延迟测试报告 |
| 准确率验证 | 在广西边境实测数据集上完成端到端评估，覆盖白天/夜间/雨雾/动物等多种场景 | Accuracy ≥ 85% 的正式评测报告 |

---

### 8.2 开发组

**核心目标**：基于 uni-app 完成 APP 核心四大页面（登录、态势一张图/预警工作台、一键取证）开发，实现离线缓存与水印取证基础功能。

| 工作项 | 具体内容 | 交付物 |
|-------|---------|--------|
| 登录与身份认证 | 统一身份认证（JWT），角色权限控制（frontline / investigator / commander），适配离线 Token 缓存 | 登录页 + 权限验证逻辑 |
| 态势一张图（GIS） | 集成 uni-app 地图，渲染五大战区图层（红外点位/振动围栏/巡逻路线/走私热区），底部三个战术按钮（SOS/一键处置/任务确认） | GIS 全屏地图页（支持图层切换） |
| 预警工作台 | 三分类 Tab（走私预警/环保预警/食品药品），筛选栏，动态字段卡片，预警详情弹窗，分类独立字段渲染 | Alert Center 完整功能页 |
| 一键取证 | 拍照/录像入口，自动追加水印（时间戳 + GPS + 警号），生成防伪校验码（EV-HASH-TS），离线本地缓存与断点续传 | 取证功能页 + 水印/校验模块 |
| 离线缓存 | 使用 uni storage 缓存预警列表和待提交取证文件，网络恢复后自动同步上传 | 离线队列管理模块 |

---

### 8.3 硬件组

**核心目标**：采购核心传感器（红外 + 振动 + 可见光，选平价款），搭建最小感知硬件系统，实现数据采集并传输至 APP。

| 工作项 | 具体内容 | 交付物 |
|-------|---------|--------|
| 传感器选型采购 | 红外热像模组（分辨率 ≥ 160×120，可选 FLIR Lepton 级平价款）、振动传感器（压电陶瓷或 MEMS，加速度量程 ±16g）、可见光摄像头（1080P，TTL/UART 接口） | 采购清单 + 采购合同 |
| 最小感知系统搭建 | 以 ESP32 / STM32 为核心控制单元，完成三路传感器信号采集电路设计与 PCB 绘制 | 硬件原理图 + PCB 文件 |
| 数据传输通道 | 传感器节点通过 MQTT over WiFi（或 LoRa 中继）将数据上报至云端 API；APP 端 WebSocket 实时接收并推送给算法推理模块 | 通信协议文档 + MQTT Topic 清单 |
| 边缘推理对接 | 树莓派/jetson nano 等边缘网关接收融合后数据，运行算法推理后生成预警事件写回消息队列 | 边缘推理接口文档 |

---

### 8.4 设计组

**核心目标**：优化 APP 暗色模式与大按钮交互，贴合民警戴手套操作习惯，确保一线高可用性。

| 工作项 | 具体内容 | 交付物 |
|-------|---------|--------|
| 暗色模式规范 | 统一深色配色体系（深蓝黑底 #060A14 + 全息蓝 #00D4FF 主色），确保文字对比度 ≥ 4.5:1，满足强光户外可读性 | 设计规范文档（色板/字体/间距） |
| 大按钮战术交互 | 按住确认机制（SOS 1.5s / 处置/确认 1.0s），触控热区外扩 16rpx，支持戴手套盲操作，触觉反馈（vibrateLong） | TacticalButton 组件规范 |
| 信息密度优化 | 预警卡片按分类（走私/环保/食药）动态渲染字段，无数据格不显示；关键信息优先，风险评分环突出；操作按钮精简为"忽略/处置"双按钮 | 各页面信息结构规范文档 |
| 取证页水印设计 | 水印样式规范（半透明背景 + 时间戳/经纬度/警号三行叠加），视频水印叠加方案 | 取证 UI 设计稿（Figma） |
| 响应式适配 | 横屏地图大屏优先，平板双列卡片布局，手机竖屏单列紧凑排版 | 多端响应式设计说明 |

### 8.5 组间协作接口

| 接口方向 | 内容 | 格式 |
