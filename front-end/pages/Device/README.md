# 传感器网络 - Device

## 📋 项目概述

"传感器网络"是**热眼擒枭**边境巡逻系统的**系统透明化与自愈引导**页面。在边境极端环境下，设备故障是常态，该页面不仅告诉巡逻员"设备坏了"，更要通过**诊断化视觉**告诉他们"坏在哪里"以及"是否影响当前的逻辑链路"。

---

## 🎨 设计理念

### 核心设计原则
1. **诊断化视觉** - 通过拓扑连线发现链路故障
2. **自愈引导逻辑** - 自动置顶需要检查的设备
3. **弱网容错设计** - 同步进度显性化，缓解心理焦虑
4. **健康脉动动画** - 用动效直观展示设备状态

### 三层布局架构

```
┌─────────────────────────────────────┐
│  顶部：系统健康指数与同步概览        │
│  ├─ 健康仪表盘（环形图显示在线率）   │
│  └─ 数据同步状态带（实时滚动显示）   │
├─────────────────────────────────────┤
│  中部：拓扑态势图                    │
│  ├─ 边缘节点（中心散射）             │
│  ├─ 传感器节点（环形分布）           │
│  ├─ 连接线（流动动画）               │
│  └─ 健康脉动（心跳扩散）             │
├─────────────────────────────────────┤
│  下部：离线资源管理                  │
│  ├─ 地图包列表（按地理区块）         │
│  └─ 一键清理（释放存储空间）         │
└─────────────────────────────────────┘
```

## 🚀 核心功能

### 1. 系统健康指数 (Global Status)

#### 健康仪表盘
- **环形进度图**显示系统健康分数（0-100%）
- 颜色编码：
  - 🟢 绿色：健康分数 ≥ 80%
  - 🟠 橙色：健康分数 50-80%
  - 🔴 红色：健康分数 < 50%

#### 设备统计
- **在线设备**：绿色边框
- **预警设备**：橙色边框
- **离线设备**：灰色边框

### 2. 数据同步状态带 (SyncManager)

**实时滚动显示**：
- ✅ **数据已同步** - 绿色背景
- 🔄 **正在同步** - 橙色背景 + 进度条
  - "正在同步：3条取证视频 (45%)..."
- ⚠️ **离线模式** - 红色背景
  - "待同步：5条证据已加密保存"

**设计说明**：将同步进度显性化，能极大缓解一线人员"怕数据没传上去"的心理焦虑。

### 3. 拓扑态势图 (TopologyView)

#### 核心视觉
- **边缘节点**：中心位置，紫色渐变
- **传感器节点**：环形分布，根据类型显示不同颜色
- **连接线**：虚线流动动画

#### 健康脉动动画
- **正常**：淡绿色光圈缓慢扩散（类似心跳）
- **预警**：黄色呼吸灯闪烁
- **离线**：灰色暗淡状态，连接线断裂

#### 交互
点击拓扑节点，弹窗显示：
- 设备名称
- 最后活跃时间
- 电量和信号强度
- 地理位置

### 4. 设备列表 (DeviceStatCard)

#### 自动排序
设备按优先级自动排序：
1. **离线设备**（最高优先级）
2. **预警设备**（低电量/弱信号）
3. **在线设备**

#### 建议检查标签
当巡逻员靠近故障设备时（基于 GPS），该设备会：
- 自动置顶
- 显示"建议顺路检查"红色标签
- 利用人力流动实现系统的"自检自愈"

#### 显示信息
- 设备图标（渐变色背景）
- 设备名称和 ID
- 最后活跃时间
- 电量百分比（颜色编码）
- 信号强度 RSSI（颜色编码）

### 5. 离线资源管理

#### 地图包列表
按地理区块划分：
- 磨憨段
- 勐腊段
- 景洪段

每个地图包显示：
- 名称和大小
- 更新时间
- 下载状态（已下载/可下载/下载中）

#### 一键清理
- 清理已上传的缓存数据
- 释放手机存储空间
- 显示存储使用情况

## 🎯 设计说明

### 为什么这样布局？

#### 1. 诊断化视觉 (Diagnostic UI)
传统的列表式展示很难发现"链路故障"。通过拓扑连线，巡逻员可以一眼看出：由于 A 处的边缘计算节点断网，导致下游的 5 个气味探头全部失效。这比逐条查看设备列表高效得多。

#### 2. "自愈"引导逻辑
页面不仅显示状态，还具备引导功能。例如，当巡逻员靠近一个故障设备（基于蓝牙或 GPS 发现），该设备在列表中的顺序会自动置顶，并显示"建议顺路检查"提示，利用人力流动实现系统的"自检自愈"。

#### 3. 弱网容错设计
`SyncManager.vue` 是该页面的灵魂。在边境，网络是奢侈品。将同步进度显性化，能极大缓解一线人员"怕数据没传上去"的心理焦虑。

## 🔧 核心算法

### 1. 计算健康分数
```javascript
calculateHealthScore(devices)
```
综合考虑：
- 设备状态（离线/预警/在线）
- 电量水平
- 信号强度

### 2. 设备优先级排序
```javascript
sortDevicesByPriority(devices)
```
排序规则：
1. 离线设备优先
2. 预警设备次之
3. 相同状态下，低电量优先

### 3. 判断是否需要检查
```javascript
shouldCheckDevice(device, currentLocation)
```
条件：
- 设备离线或电量 < 20%
- 距离当前位置 < 500米

### 4. 生成健康报告
```javascript
generateHealthReport(devices)
```
返回：
- 总设备数
- 在线/预警/离线设备数
- 低电量设备列表
- 弱信号设备列表
- 关键设备列表
- 健康分数

## 🔄 交互流程

### 查看设备详情
1. 点击拓扑图中的设备节点
2. 弹窗显示设备详细信息
3. 包括状态、电量、信号、最后活跃时间

### 下载离线地图
1. 点击地图包的"下载"按钮
2. 显示下载进度条
3. 下载完成后显示"已下载"标签

### 清理缓存
1. 点击"清理已上传缓存"按钮
2. 确认清理操作
3. 释放存储空间并更新显示

### 自动刷新
- 每 30 秒自动刷新设备状态
- 手动点击刷新按钮立即更新
