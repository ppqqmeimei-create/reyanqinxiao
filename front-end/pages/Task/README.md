# 任务执行页 - Task

## 📋 项目概述

"任务执行页"是**热眼擒枭**边境巡逻系统的**实战核心页面**，当巡逻员从"情报获取"转入"现场拦截"时使用。设计核心在于**极度简化、交互热区极大化、反馈明确化**，适应快速移动、丛林穿梭或高压对峙状态。

---

## 🎨 设计理念

### 核心设计原则
1. **盲操作与触感引导** - 通过震动反馈确认操作，无需长时间盯屏
2. **认知负荷最小化** - 只显示"我要去哪"和"我要怎么录"
3. **证据链瞬时锚定** - 拍照瞬间即数据同步时刻
4. **防误触设计** - 侧滑确认代替点按，防止意外关闭任务

### 三层布局架构

```
┌─────────────────────────────────────┐
│  顶部：任务态势与时效                │
│  ├─ 任务紧急度（背景闪烁）           │
│  ├─ 距离与方位（巨大字体 + 箭头）    │
│  └─ 时效倒计时（预计接触时间）       │
├─────────────────────────────────────┤
│  中部：多模态一键取证区              │
│  ├─ AR 取证窗口（相机预览）          │
│  ├─ 多模态仪表盘（气味/震动/红外）   │
│  ├─ 一键录证大按钮（右侧悬浮）       │
│  └─ 雷达导航（方位指引）             │
├─────────────────────────────────────┤
│  底部：防误触闭环操作                │
│  ├─ 断网暂存箱（显示待同步数量）     │
│  └─ 侧滑确认（长距离滑动完成任务）   │
└─────────────────────────────────────┘
```

## 🚀 核心功能

### 1. 任务态势与时效 (Mission Header)

#### 任务紧急度
- 背景根据风险等级（红/橙）闪烁
- 高危任务：红色脉冲动画
- 预警任务：橙色流动动画

#### 距离与方位
- **巨大字体显示距离**（120rpx）
- **动态箭头**指向拦截点，实时旋转
- **发光效果**增强可读性

#### 时效倒计时
- 基于逻辑推理显示"预计接触时间"
- 小于30秒时触发紧急模式
- 营造紧迫感，提升响应速度

### 2. 拦截雷达 (InterceptRadar)

**雷达可视化**：
- 4层同心圆（100m、300m、500m、1000m）
- 扫描线动画（4秒一圈）
- 中心点（我）：绿色脉冲
- 目标点（目标）：红色闪烁 + 扩散圈

**方位指示**：
- 东南西北标注
- 实时计算方位角
- 显示方位文本（如"东北 45°"）

### 3. 多模态取证 (MultiEvidence)

#### AR 取证窗口
- 调用摄像头实时预览
- 侧边悬浮多模态仪表盘
- 支持前后摄像头切换

#### 多模态数据叠加
- **气味浓度**：进度条显示 + 百分比
- **震动强度**：波形图实时跳动
- **红外温度**：数值显示

#### 一键录证大按钮
- 位于屏幕右侧最顺手位置
- 直径 160rpx，红色渐变
- 点击瞬间：
  - 短促有力震动（类似枪机扣动感）
  - 自动抓取所有传感器数值
  - 生成带数字签名的证据包

#### 静默录音
- 开启后台录音
- 自动与任务 ID 绑定
- 录音中显示红色指示器

### 4. 元数据水印 (MetadataBadge)

**自动封装数据**：
- 时间戳（精确到秒）
- 经纬度（6位小数）
- 海拔高度
- 传感器数据摘要

**颜色编码**：
- 🔴 红色：危险值（气味≥80%，体温35-38°C）
- 🟠 橙色：警告值（气味50-80%）
- 🟢 绿色：正常值

### 5. 侧滑确认 (ActionSlider)

**防误触设计**：
- 必须向右长距离滑动（90%以上）
- 滑动过程中显示进度填充
- 滑动到80%以上显示"松手完成"
- 未滑到底自动回弹

**触觉反馈**：
- 开始滑动：短震动
- 完成任务：长震动
- 回弹：短震动

### 6. 断网暂存箱

**离线优先**：
- 显示待同步证据数量
- 加密保存在手机闪存
- 网络恢复后自动上传

**DiffSync 算法**：
- 碎片化上传关键摘要
- 即使手机损坏也能恢复数据

## 🎯 设计说明

### 为什么这样布局？

#### 1. 盲操作与触感引导 (Haptic First)
在极端环境下，巡逻员无法长时间盯着屏幕。点击取证按钮时，APP 会反馈一段**短促有力的震动**（类似枪机扣动感），确认操作成功。

#### 2. 认知负荷最小化 (Cognitive Efficiency)
界面不显示任何非核心参数。只有"我要去哪"和"我要怎么录"。`MetadataBadge` 会自动将复杂的经纬度、气味数据封装，巡逻员只需看最终的红色预警即可。

#### 3. 证据链的"瞬时锚定"
通过 `useEvidence.js`，拍照的时刻即是数据同步的时刻。即使手机随后被毁或丢失，只要在有微弱信号瞬间，碎片化的加密数据包就会通过 DiffSync 算法先行上传关键摘要。

#### 4. 防误触的侧滑设计
绝对不能使用"点按"来完成任务。巡逻员必须向右长距离滑动才能触发"任务闭环"。这防止了在抓捕或奔跑中误触导致任务意外关闭。

## 🔧 核心算法

### 1. 距离计算 (Haversine 公式)
```javascript
calculateDistance(point1, point2)
```
精确计算地球表面两点间距离（米）

### 2. 方位角计算
```javascript
calculateBearing(point1, point2)
```
计算从当前位置到目标的方位角（0-360度，北为0度）

### 3. 预计到达时间
```javascript
calculateETA(distance, speed = 3)
```
根据距离和速度计算预计到达时间

### 4. 拦截点计算
```javascript
calculateInterceptPoint(targetPos, targetVelocity, patrolPos, patrolSpeed)
```
根据目标移动方向和速度，计算最佳拦截位置

### 5. 证据采集
```javascript
captureEvidence(data)
```
- 生成唯一证据ID
- 保存到本地存储
- 尝试上传（有网络时）
- 返回证据对象

### 6. 离线同步
```javascript
syncOfflineEvidence()
```
批量上传所有未同步的证据

## 🔄 交互流程

### 任务执行流程
1. 从预警页面接收任务数据
2. 开始实时定位和导航
3. 显示距离、方位、倒计时
4. 巡逻员根据雷达指引前进
5. 到达现场后进行取证
6. 侧滑完成任务

### 取证流程
1. 点击"一键录证"按钮
2. 触发震动反馈
3. 自动抓取传感器数据
4. 生成证据包并保存
5. 尝试上传（有网络时）
6. 显示暂存箱数量

### 完成任务流程
1. 距离小于50米时启用侧滑
2. 向右滑动滑块
3. 滑动到90%以上
4. 松手触发完成
5. 长震动反馈
6. 提交任务并返回
